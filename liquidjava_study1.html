Exercises -
Understand the error in the code through the message

In these exercises you will be asked to look at the code and the error message and understand what is the error prompted in the message.
We aim to get a picture of how you understand the error through the error message. Therefore we ask you to:

  1. Try to understand what is the error in the code related to the incorrect use of refinements
  2. Talk aloud describing each step and thought you have towards understanding the error

public static int safeDivide(int num, 
@Refinement("den != 0") int den) {
    return num/den;
}
    
public static void main(String[] args) {
    @Refinement("0 <= x && x <= 100")
    int x = safeDivide(6, 10-10);
}

What would fix it?


The method safeDivide is invoked with the second argument with the value zero which goes against the refinement.
The solution would be to change the second argument to a different value (different than zero).

@StateSet({"unconnected", "bind", 
            "connected", "closed"})
public class Socket {
    
@StateRefinement(to="unconnected(this)")
public void Socket();

@StateRefinement(from="unconnected(this)",
                    to="bind(this)")
public void bind(SocketAddress add);
    
@StateRefinement(from="bind(this)",
                    to="connected(this)")
public void connect(SocketAddress add);
    
    
@StateRefinement(from="connected(this)")
public void sendUrgentData(int n);
    
@StateRefinement(to="closed(this)")
public void close();

}
